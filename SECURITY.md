# 🔒 Security Policy

Thank you for helping keep **Awesome Python Projects Community** secure.

We appreciate responsible disclosure of security vulnerabilities.

---

# 📌 Supported Versions

| Version | Supported |
|---------|-----------|
| v1.x | ✅ Yes |
| Older Versions | ❌ No |

---

# 🚨 Reporting a Vulnerability

If you discover a security vulnerability, **please do not create a public GitHub Issue**.

Instead, contact the repository maintainer through GitHub before publicly disclosing the issue.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected files or project
- Screenshots (if applicable)
- Suggested fix (optional)

---

# 📋 What to Report

Examples include:

- Malicious code
- Remote code execution vulnerabilities
- Dependency vulnerabilities
- Sensitive information accidentally committed
- Security issues in contributed projects
- Unsafe scripts or automation

---

# ⏳ Response Time

We aim to:

- Acknowledge your report within **7 days**.
- Investigate and verify the issue.
- Release a fix as soon as reasonably possible.

Response times may vary depending on the complexity of the issue.

---

# 🔐 Responsible Disclosure

Please:

- Give maintainers time to investigate.
- Do not publicly disclose vulnerabilities before they are addressed.
- Avoid exploiting vulnerabilities beyond what is necessary to demonstrate the issue.

Responsible disclosure helps protect all users of this repository.

---

# 🙏 Thank You

We appreciate everyone who helps improve the security of this project.

Your responsible reporting helps keep our community safe.

Thank you for supporting open-source development.
