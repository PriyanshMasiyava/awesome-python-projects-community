# 🤝 Contributing to Awesome Python Projects Community

First of all, thank you for your interest in contributing to **Awesome Python Projects Community**! 🎉

We welcome contributions from developers of all skill levels. Whether you're a beginner writing your first Python script or an experienced developer building advanced applications, your contribution is valuable to our community.

Our goal is to build one of the largest open-source collections of Python projects where everyone can learn, collaborate, and grow together.

You can contribute by:

- 🐍 Adding new Python projects

- 🐞 Fixing bugs

- ✨ Adding new features
- 🎨 Improving code quality
- 🧪 Adding tests
- 💡 Suggesting new project ideas

# 📂 Repository Structure

Please add your project to the appropriate folder.

```
awesome-python-projects-community/
│
├── Python/
│   ├── 01_Beginner/
│   ├── 02_Intermediate/
│   └── 03_Advanced/
│
├── assets/
├── docs/
└── README.md
```

### Choose the correct category

| Difficulty | Folder |
|------------|----------------------------|
| 🟢 Beginner | `Python/01_Beginner/` |
| 🟡 Intermediate | `Python/02_Intermediate/` |
| 🔴 Advanced | `Python/03_Advanced/` |

Please do **not** place projects outside these folders.

# 📋 Before You Start

Before contributing, please:

- ⭐ Star the repository
- 🍴 Fork the repository
- 📖 Read the README
- 📖 Read this CONTRIBUTING guide
- Search existing Issues and Pull Requests to avoid duplicates

---

# 🚀 How to Contribute

## Step 1 — Fork the Repository

Click the **Fork** button on GitHub.

---

## Step 2 — Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/awesome-python-projects-community.git
```

---

## Step 3 — Create a New Branch

```bash
git checkout -b feature/project-name
```

Example:

```bash
git checkout -b feature/calculator-project
```

---

## Step 4 — Add Your Project

Place your project inside the correct directory.

Example:

```
Python/
└── 01_Beginner/
    └── Calculator/
```

---

## Step 5 — Project Structure

Every project should follow this structure whenever applicable.

```
Project_Name/
│
├── README.md
├── main.py
├── requirements.txt
├── assets/
└── images/
```

If your project does not require `requirements.txt`, you may omit it.

---

## Step 6 — Commit Your Changes

```bash
git add .

git commit -m "Add Calculator Project"
```

Use meaningful commit messages.

Examples:

```
Add BMI Calculator

Fix Weather App bug

Improve README

Update Documentation
```

---

## Step 7 — Push Your Branch

```bash
git push origin feature/project-name
```

---

## Step 8 — Create a Pull Request

Open a Pull Request describing:

- What you added
- Project category
- Libraries used
- Any additional information

Our maintainers will review your contribution.

---

# 📦 Project Requirements

Your project should:

- Be written in Python
- Run without errors
- Include clean and readable code
- Include a README.md
- Include requirements.txt (if external libraries are used)
- Use meaningful file names
- Follow the repository folder structure

---

# 📁 Folder Naming Convention

Please use descriptive names.

✅ Good Examples

```
Calculator

BMI Calculator

Expense Tracker

Weather App

Chat Application
```

❌ Avoid

```
Project1

Task2

PythonApp

Demo
```

---

# 📝 README Guidelines

Each project should include a README.md explaining:

- Project Name
- Description
- Features
- Installation
- Usage
- Requirements
- Screenshots (optional)
- Author (optional)

---

# 🎯 Coding Guidelines

Please:

- Follow Python best practices
- Write clean and readable code
- Use meaningful variable names
- Add comments where necessary
- Remove unused files
- Remove unnecessary print statements

---

# 🚫 What Not to Submit

Please do not submit:

- Copied projects without attribution
- Malicious code
- Spam
- Broken or incomplete projects
- Unrelated files
- Binary files unless necessary

---

# ✅ Pull Request Checklist

Before submitting your Pull Request, ensure:

- Project works correctly
- Folder is in the correct category
- README.md included
- requirements.txt included (if required)
- No unnecessary files
- Code follows repository guidelines
